Run the same matching logic used when opportunities are published and in the pipeline. View results by model (one-way, two-way barter, consortium, circular).

How matching works

One-way

For a Need post (request), the system finds Offer posts that match (skills, budget, timeline, location). Default when viewing a request.

Two-way (barter)

When a creator has both a Need and an Offer, the system finds other creators whose Offer satisfies this creator’s Need and whose Need is satisfied by this creator’s Offer.

Consortium

A lead Need with defined roles (e.g. Financial partner, Construction expertise, Equipment fleet) gets one best Offer per role from distinct partners.

Circular

The system finds cycles of length 3 or more (e.g. A→B→C→A) where each participant’s Offer satisfies the next participant’s Need.
